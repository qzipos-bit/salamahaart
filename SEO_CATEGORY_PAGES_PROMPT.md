# Промт: отдельные страницы категорий (профессиональный SEO)

Используйте этот текст как ТЗ для ИИ или подрядчика при доработке категорий.

---

Ты — senior SEO и фронтенд-разработчик (Next.js App Router). Нужно вынести каждую товарную категорию на **отдельный канонический URL** (чистые пути без `?cat=`), с уникальным контентом и технической базой под поиск.

## Цели

1. **Один URL — один интент**: страница категории отвечает на запросы вроде «столы из эпоксидной смолы», «часы из смолы на заказ» и т.п.
2. **Исключить дубли**: старые адреса с `?cat=` отдают **301** на новый путь.
3. **Уникальные title, description, H1** и **2+ абзаца** вводного текста на категорию (без копипаста с главной каталога дословно).
4. **Иерархия заголовков**: один H1; карточки товаров — H2 (или H3 под локальным H2 «Товары в категории» — на выбор, но последовательно).
5. **Внутренняя перелинковка**: хлебные крошки (Главная → Каталог → Категория), ссылки на соседние категории и на общий `/catalog`.
6. **Structured data**: `BreadcrumbList` + `ItemList` (или `CollectionPage` + элементы) для списка товаров на странице категории; глобально уже есть `Organization`.
7. **Индексация**: страницы в **sitemap.xml**; `robots.txt` не блокирует каталог.
8. **Товары**: у каждого товара в данных явное поле **категория**; фильтрация списка на странице категории по этому полю.
9. **Пустая категория**: осмысленный текст + CTA (каталог / WhatsApp), без soft-404.
10. **Конфликт маршрутов**: slug категории и slug товара **не должны пересекаться**; при одном динамическом сегменте — сначала проверка «категория», затем «товар», иначе 404.

## Что сдать в ответе / в коде

- Список URL категорий и таблица: title, description, H1, аннотация для клиента.
- Компоненты/страницы, middleware/редиректы, обновлённые ссылки с главной.
- Проверка: Lighthouse/валидатор schema.org, ручной просмотр исходника HTML (мета и JSON-LD на месте).

## Критерии качества

- Title ~50–60 символов, description ~140–160, без переспама.
- Текст читается естественно, ключи вписаны в смысл.
- Мобильная вёрстка и те же данные, что и на десктопе (без сокрытия контента для робота).

## Реализованные тематические URL (SEO-лендинги)

Данные и тексты: `src/lib/seo-catalog-landings.ts`. Общий вид страницы: `src/components/catalog/seo-catalog-landing-view.tsx`. Маршруты — отдельные `page.tsx` под каждый путь. В `src/app/sitemap.ts` добавлены эти URL (при заданном `NEXT_PUBLIC_SITE_URL`).

| URL | Назначение |
|-----|------------|
| `/catalog/dekorativnye-stoliki/breakfasttable` | Столик для завтрака в постель |
| `/catalog/tarelki-i-blyuda-iz-epoksidnoj-smoly` | Тарелки и посуда из смолы |
| `/catalog/podnosy-iz-epoksidnoj-smoly` | Подносы |
| `/catalog/chasy-kartiny/chasy-iz-smoly` | Часы |
| `/catalog/chasy-kartiny/kartiny-iz-smoly` | Картины и панно |
| `/catalog/servirovochnye-doski-iz-dereva-i-smoly` | Сервировочные доски и менажницы |

Карточки товаров на лендинге задаются массивом `productSlugs` в том же файле; пустой массив — блок с приглашением в WhatsApp.
