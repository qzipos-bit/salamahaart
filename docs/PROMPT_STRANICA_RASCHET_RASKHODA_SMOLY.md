# Промпт: отдельная страница «Расчёт расхода смолы»

Используйте этот документ как единое ТЗ для реализации **автономной страницы** в существующем Next.js-проекте салона/магазина (Tailwind, `globals.css`, шрифты DM Sans и т.д.). Визуальная и логическая опора — приложенные скриншоты рабочего калькулятора.

---

## 1. Назначение и аудитория

- Страница помогает посетителю **оценить массу эпоксидной смеси** (и доли компонентов A/B) для заливки или покрытия, либо **пересчитать известный вес** в компоненты по пропорции.
- Язык интерфейса: **русский** (все подписи, ошибки, FAQ). Заголовки и пояснения без смешения языков (в референсе было «Seleziona…» — **заменить на корректный русский**).

---

## 2. Маршрут и оболочка

- Отдельный маршрут, например: `/raschet-raskhoda-smoly` или `/instrumenty/raschet-smoly` (согласовать с навигацией сайта).
- Подключить общий **layout** (шапка/подвал), как на остальных страницах.
- **SEO**: `title`, `description`, один H1 — «Расчёт расхода смолы» (или вариант с брендом). Структурированные данные по желанию (`WebPage` + `FAQPage` если FAQ статичен).

---

## 3. Общий визуал (как на скринах)

- **Фон страницы**: светлый крем/беж, гармонирующий с `--bg` / `--cream` проекта; при необходимости чуть теплее для «калькуляторной» зоны.
- **Карточки**: белый фон, скругление (~12–20px в духе `--radius`), лёгкая тень или обводка.
- **Акцент выбора**: зелёная рамка + лёгкий зелёный фон у активной карточки (можно взять `--green` / производные, не обвер-токсичный неон).
- **Иконки**: упрощённые «3D» пиктограммы (капля на блоке, цилиндр, тонкие пластины, волна и т.д.) — SVG или статика, единый стиль (синие/голубые плоскости на скрине).
- **Типографика**: sans, жирные заголовки шагов, вторичный текст серее.
- **Адаптив**: на узком экране колонки **стекаются**: сначала шаги 1–2, затем ввод и результат; FAQ на всю ширину снизу.

## 4. Структура макета

### Колонка слева (настройка режима)

#### Блок «1. Выберите тип калькулятора»

Две крупные **переключаемые карточки** (как radio):

| Ключ | Текст | Иконка (смысл) |
|------|--------|----------------|
| `volume` | Сколько смолы нужно для заливки? | Капля на 3D-параллелепипеде |
| `components` | Сколько компонента A и/или B нужно? | Схема смешивания A и B |

Активная — зелёная обводка и лёгкий зелёный тинт.

#### Блок «2» — зависит от выбора в п.1

**Если выбран `volume` — «2. Выберите тип заливки»**  
Сетка из **5 вариантов** (последний — на всю ширину строки):

1. Прямоугольная заливка — «толстый» прямоугольный объём.  
2. Круглая заливка — цилиндр.  
3. Прямоугольное покрытие — тонкая пластина.  
4. Круглое покрытие — тонкий диск.  
5. Заливка неправильной формы — волнистый/органический объём.

**Если выбран `components` — «2. Выберите тип расчёта»**  
Краткое пояснение под заголовком:  
*«Расчёт количества каждого компонента, если известен вес готовой смеси или одного из компонентов».*

Три **вертикальные** карточки:

1. Я знаю только общее количество (A+B), которое мне нужно.  
2. Я знаю только вес компонента A, сколько нужно компонента B?  
3. Я знаю только вес компонента B, сколько нужно компонента A?

(На референсе ошибочно был итальянский заголовок — **везде русский**.)

### Колонка справа (ввод и результат)

Порядок блоков сверху вниз:

#### «Введите размеры» (или эквивалент по режиму)

- Постоянное примечание: **«Примечание: длина и ширина в см, толщина в мм»** — единицы **см** и **мм** визуально выделить (например красным/акцентом), как на скрине.
- Поля **зависят от типа заливки** (`volume`):

| Тип заливки | Поля |
|-------------|------|
| Прямоугольная заливка | Длина (см), Ширина (см), Толщина (мм) |
| Круглая заливка | Диаметр (см), Толщина (мм) |
| Прямоугольное покрытие | Длина (см), Ширина (см), Толщина (мм) |
| Круглое покрытие | Диаметр (см), Толщина (мм) |
| Неправильная форма | **Вес воды (г)** — одно поле |

- Суффиксы единиц справа от полей: `см`, `мм`, `г`.
- Для режимов **покрытия** (прямоугольное/круглое): жёлтый/мягкий info-блок:  
  *«Минимальная рекомендуемая толщина покрытия смолой: 1,5–2 мм»* (допускается предзаполнение толщины `1,5` с учётом локали).
- Для **неправильной формы**: информационный блок (голубой/нейтральный) с инструкцией: поставить форму на весы, обнулить, залить водой до уровня заливки, внести **массу воды в граммах** (объём в мл ≈ масса воды в г).

#### «Введите пропорции смолы»

- Текст-помощник: пропорции обычно на этикетке/у продавца, пример **A:B = 100:60**, ссылка «Подробнее…» (якорь на секцию FAQ или отдельный абзац).
- Два поля рядом: **Компонент A**, **Компонент B** (числа, допускают десятичные — см. парсинг).

#### «Результат»

- В режиме **`volume` после расчёта объёма**: три визуальных итога (как на одном из скринов):
  - **Всего** — светло-зелёная плашка, справа жирное значение «N г».
  - **Компонент A** — светло-синяя/голубая плашка, «N г».
  - **Компонент B** — светло-розовая плашка, «N г».
- В режиме **`components`**: по смыслу подтипа — одна или две строки (например только «Всего» при вводе суммы A+B; при вводе только A — показать вычисленные A, B и сумму). Визуально — та же зелёная плашка минимум для итога, остальное — по ТЗ дизайна.

При пустых или невалидных данных показывать **0 г** или «—», без ложных дробей.

## 5. Логика расчётов (обязательно явно в коде)

### Общие обозначения

- Пропорция: части `partA` и `partB` (например 100 и 60).  
  Доля A в смеси: `partA / (partA + partB)`, доля B с симметрией.
- Парсинг чисел: поддержка **запятой и точки** (`1,5` / `1.5`).
- Объём в **см³**:
  - Прямоугольник: \(V = L \times W \times H\), где \(L,W\) в см, \(H\) — толщина в мм **делить на 10**.
  - Цилиндр (заливка): \(V = \pi \times (d/2)^2 \times H\) (d в см, H в см = мм/10).
  - Покрытия — те же формулы, **H** — малая толщина в см.

### Масса смолы из объёма

- Перевод объёма в массу: использовать **константу плотности** смеси (например **1,1 г/см³**), вынести в один конфиг/константу с комментарием, что это усреднение (реальная плотность зависит от бренда).
- `massTotal_g = volume_cm3 * DENSITY_G_PER_CM3`.

### Неправильная форма

- Считать, что **масса воды (г) ≈ объём в мл ≈ объём в см³** для воды; для смолы затем: `massTotal_g = water_g * DENSITY_G_PER_CM3` (как упрощение «объём формы = объём воды»). В подсказке явно написать метод с весами.

### Разбивка на A и B

- `massA = massTotal * partA / (partA + partB)`  
- `massB = massTotal * partB / (partA + partB)`  
- Округление: разумное для UI (например 1 знак после запятой или целые граммы — указать в константе).

### Режим «только компоненты» (известен один из весов)

Реализовать три подрежима:

1. **Известна сумма A+B (г)** → из суммы и пропорций вычислить A и B.  
2. **Известен только A** → \(B = A \times (partB/partA)\), сумма = A + B.  
3. **Известен только B** → \(A = B \times (partA/partB)\), сумма = A + B.

Защита от деления на ноль и от отрицательных/пустых пропорций — дружелюбные сообщения под полями или tooltip.

## 7. Интерактив и UX

- Пересчёт **on change** / on blur полей (debounce опционален).
- Сохранение выбранных карточек в **URL query** или `sessionStorage` — по желанию (в промпте можно заложить «без персистенса v1»).
- Фокус, `label` + `htmlFor`, контраст, клавиатура для карточек-«radio» (`role="radio"`, `aria-checked` или нативный `input type="radio"` скрытый).
