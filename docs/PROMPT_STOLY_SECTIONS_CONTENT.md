# Промпт: наполнение раздела «Столы» (pillar + фасеты)

Используйте этот текст **целиком** как ТЗ для Cursor, SEO-копирайтера или дизайнера контента.  
К нему прикладывается **`src/lib/data/stoly-content.json`** — единый источник текстов и медиа для всех URL ветки `/catalog/stoly`.

---

## Роль

Ты — **senior SEO-копирайтер + арт-директор студии мебели из смолы + фронтенд (Next.js App Router, Tailwind 4)** в проекте **Salamaha Fine Art** (Кrasnodar, заказ, доставка по РФ).

Задача: превратить ветку **«Столы»** из «страницы с одним абзацем и серых карточек „на заказ“» в **убедительный коммерческий раздел** — как у сильных студий river table: визуал, сценарии, доверие, понятный путь к заказу.

**Не трогать без запроса:** meta `title` / `description` (уже согласованы с SEO). Можно уточнять `h1` только если явно указано в брифе.

Стек: Next.js 16, `StolyPageView`, `stoly-content.json`, `ProductCard`, палитра `globals.css`, шрифты Cormorant + Manrope.

---

## Диагноз: почему сейчас «беспонтово»

| Проблема | Где видно | Что чувствует пользователь |
|----------|-----------|----------------------------|
| **Один плотный абзац intro** | Все 8 страниц | «Прочитал Wikipedia, не понял, что заказать» |
| **Портфолио = текст без фото** | P1/P2/P3 фасеты | «Это вообще существует или просто SEO?» |
| **Фасеты на pillar — список ссылок** | `/catalog/stoly` | Нет превью, нет «крючка» под каждый интент |
| **Один товар `stol-river`** | pillar + zhurnalnyj | Вся ветка держится на одной карточке |
| **FAQ есть, но без контекста** | Все страницы | Ответы сухие, нет связи с процессом студии |
| **Нет блока «как заказать»** | Везде | Не снижает страх первого заказа |
| **Нет социального доказательства** | Везде | Нет этапов работ, упаковки, отзывов, «до/после» |

**Цель не «больше текста ради SEO»**, а **конверсия + доверие + уникальный интент каждого URL**.

## Целевая структура страницы (обязательные блоки)

Каждая страница (pillar и фасет) после наполнения должна содержать **видимые секции в HTML** в таком порядке:

| # | Блок | H-тег | Объём | Зачем |
|---|------|-------|-------|-------|
| 1 | Hero | **H1** (уже есть) | + **подзаголовок** 1 строка | Сразу интент и выгода |
| 2 | Lead intro | — | 2–3 абзаца или lead + bullets | Раскрыть H1, не дублировать meta description |
| 3 | **Highlights** (3–4 USP) | — | иконка/маркер + заголовок + 1 строка | Сканируемость на мобиле |
| 4 | Подборки / витрина | **H2** | см. ниже | Коммерческое ядро |
| 5 | **Как мы делаем** (процесс) | **H2** | 4–5 шагов | Снижает страх заказа |
| 6 | FAQ | **H2** + **H3** на вопрос | 3–7 пар | SEO + возражения |
| 7 | CTA | — | WhatsApp + «пришлите размеры» | Конверсия |
| 8 | Перелинковка | **H2** | 2–5 ссылок | Вес внутри ветки |

**Pillar дополнительно:** блок **«Подборки по типу»** — не список ссылок, а **карточки с фото, 1 строкой интента и ключом** (см. JSON `facetCards`).

## Тон и стиль (Salamaha)

- **Спокойная премиальность**, не маркетинговый крик. Без «лучшие», «№1», «уникальное предложение».
- **Конкретика:** породы, сроки в неделях, «от … ₽» где честно возможно.
- **Вы — мастерская на заказ**, не маркетплейс: «пришлите размеры», «согласуем в WhatsApp», «фото этапов».
- **Ключи вписаны в смысл**, не списком. Один primary_keyword на страницу (поле уже есть).
- **Не дублировать** intro pillar дословно на фасетах — у каждого URL свой угол.

---

## Критерии приёмки (Definition of Done)

Страница считается **готовой**, если:

- [ ] ≥ **350 слов** видимого текста (pillar ≥ 600).
- [ ] Есть **heroSubtitle** + **highlights** (3+).
- [ ] Витрина: **минимум 1 фото** (товар или portfolio с `image`).
- [ ] Блок **processSteps** (pillar + P1 фасеты) или локальный «Как заказать» (4 шага).
- [ ] FAQ **3+** с ответами ≥ 2 предложений.
- [ ] CTA с WhatsApp + вторичное действие (калькулятор / pillar).
- [ ] Перелинковка **≥ 2** релевантных URL.
- [ ] JSON валиден, `npm run build` проходит.
- [ ] JSON-LD: CollectionPage + FAQPage без противоречий тексту на странице.

## Порядок работ (для Cursor / команды)

1. **Расширить типы** в `stoly-pages.ts` и рендер в `stoly-page-view.tsx` под новые поля JSON.
2. **Заполнить контент** по таблицам выше — сначала **P1** (pillar, iz-sleba, s-podsvetkoj).
3. **Загрузить фото** в `/public/stoly/` (см. чеклист).
4. **P2** после P1: zhurnalnyj (усилить `stol-river`), kuhonnyj, s-cvetami.
5. **P3** когда есть фото больших/письменных столов.
6. Прогнать Lighthouse mobile + проверить исходник HTML (H1 один, FAQ в DOM).

## Быстрый win без ожидания фото (если съёмка позже)

Допустимый **временный** апгрейд до появления `/public/stoly/`:

1. Разбить intro на **2–3 абзаца** + bullets highlights (текст уже сильно улучшит страницу).
2. На pillar — **facetCards** с существующими `/category-stoly.webp` и разными crop (не идеал, но лучше списка ссылок).
3. Для `stol-river` — **расширенный lead** из `catalog-product-content.json` прямо под карточкой на zhurnalnyj/pillar.
4. CTA-блок с двумя кнопками (WhatsApp + калькулятор).

**Не публиковать** отдельные P3 URL с одной текстовой карточкой «на заказ» без фото — лучше якорь на pillar.
