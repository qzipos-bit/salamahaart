# Промт: подготовка микроразметки Schema.org `Organization` для поисковых систем

Используйте как ТЗ для SEO-специалиста, разработчика или ИИ перед внедрением или аудитом.

---

## Роль

Ты готовишь **валидную JSON-LD разметку типа `Organization`** (при необходимости — `LocalBusiness` / `ProfessionalService`) для **Яндекса и Google**, без спама и с правдивыми данными.

## Обязательные правила

1. **Один граф на сайт для организации** — не дублировать десятки одинаковых блоков; достаточно **глобального блока в общем layout** (шапка сайта), чтобы он отдавался на **всех индексируемых URL**.
2. **Абсолютные URL** для `url`, `logo`, `sameAs` — только с протоколом `https://` и без цепочек редиректов на «канонический» домен.
3. **Согласованность с видимым контентом** — телефон, email, название и адрес в JSON-LD должны совпадать с футером/страницей контактов (или быть явно помечены как единственный источник правды).
4. **Тип сущности**:
   - **`Organization`** — бренд, студия, ИП без постоянной витрины.
   - **`LocalBusiness`** (подтип при необходимости) — если есть **реальный адрес приёма** и важна выдача по картам / локальному поиску; тогда добавить `geo`, часы работы при наличии.
5. **Обязательные к заполнению поля (минимум для доверия ПС)**:
   - `@context`, `@type`
   - `name`
   - `url` (канонический сайт)
6. **Настоятельно рекомендуется Google**:
   - `logo` — URL изображения (желательно квадратное/близкое к квадрату, доступное по HTTPS, без блокировки роботом)
   - при необходимости `image`
7. **Контакты** (хотя бы одно осмысленное):
   - `telephone` в формате **E.164** (`+7…`) или согласованный международный формат
   - `email` — рабочий ящик (не фиктивный `example.com` в продакшене)
   - опционально `contactPoint[]` с `ContactPoint`, `contactType`, `areaServed`, `availableLanguage`
8. **Адрес** — если указывается:
   - `PostalAddress` с `addressCountry` (ISO, например `RU`), `addressLocality`; `streetAddress` — реальная улица/дом или честная формулировка («приём по записи»), без вымышленного номера дома.
9. **`sameAs`** — только реальные официальные профили (ВК, Telegram-канал, YouTube и т.д.), без мусорных ссылок.
10. **`@id`** — стабильный URI сущности, например `https://example.com/#organization`, для связки с `WebSite`, `WebPage` в `@graph` на других страницах (по желанию).

## Чего избегать

- Выдуманные рейтинги, `aggregateRating`, если нет реальных отзывов с обоснованием.
- Несколько разных `Organization` с одним именем на одной странице без логики.
- `logo`, ведущий на 404 или на чужой домен без прав.
- Плейсхолдеры (`example.com`) в продакшене.

## Формат выдачи по промту

1. Таблица: поле → значение → источник правды на сайте.
2. Готовый блок `application/ld+json` (один объект или `@graph`).
3. Чеклист проверки: Rich Results Test / валидатор Schema.org, просмотр исходника HTML.
