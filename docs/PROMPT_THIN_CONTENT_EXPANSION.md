# Промпт: расширение «тонкого контента» на карточках товаров и страницах

Используйте этот текст **целиком** как ТЗ для Cursor / подрядчика.  
К нему прикладывается **файл с текстами** (от заказчика или SEO) — см. § «Входной файл».

---

## Роль

Ты — **senior SEO-копирайтер + фронтенд (Next.js App Router, Tailwind 4)** в проекте **Salamaha Fine Art**.  
Задача: убрать «thin content» на страницах, которые SEO-аудит пометил как бедные текстом, **без поломки вёрстки, JSON-LD и уже настроенных meta title/description**.

Стек: Next.js 16, App Router, `LandingShell`, `Container`, шрифты Cormorant + Manrope, палитра из `globals.css`.  
Не добавляй новые npm-зависимости без необходимости.

---

## Диагноз (что сейчас «тонкое»)

| Тип страницы | URL | Проблема сейчас |
|--------------|-----|-----------------|

**Meta title/description и JSON-LD уже есть** — новый видимый текст должен **совпадать по смыслу** с `product-seo.ts` / `masters-product-seo.ts` и с `Product.description` в schema.org, но **не дублировать meta description дословно** в первом абзаце.

---

## Цели

1. **Минимум уникального текста на карточке товара:** ≥ **250–400 слов** видимого контента (или структура блоков ниже с эквивалентным объёмом).
2. **Уникальность:** каждый `slug` — свой текст; запрещены массовые шаблоны с подстановкой только названия.
3. **Польза для человека:** материалы, размеры/форматы, сценарии использования, уход, сроки, доставка, как заказать — только правдивые формулировки студии (Краснодар, заказ, доставка по РФ).
4. **SEO без спама:** ключи естественно, 1 H1 на странице, подзаголовки H2/H3, списки где уместно.
5. **Единый источник данных:** тексты в **константах/TS или JSON в `src/lib/`**, не хардкод в JSX по одному абзацу на 39 файлов.
6. **Доступность:** текст в HTML (не только в картинках), логичная иерархия заголовков.

## Входной файл (от заказчика)

Заказчик прикладывает файл с контентом. Поддерживаемые форматы:

### A. JSON (предпочтительно)

```json
{
  "stol-river": {
    "lead": "Короткий абзац под ценой (2–3 предложения).",
    "sections": [
      { "heading": "Описание", "body": "Абзац или несколько через \\n\\n" },
      { "heading": "Размеры и материалы", "body": "…", "list": ["пункт 1", "пункт 2"] },
      { "heading": "Как заказать", "body": "…" }
    ],
    "specs": { "Материал": "…", "Срок": "…" }
  }
}
```

Ключ верхнего уровня = **`slug`** из `products.ts` или `masters-products.ts`.

### B. Markdown с заголовками `## slug: stol-river`

### C. Таблица CSV/XLSX (колонки: slug, block, heading, text)

**Если в файле нет блока для slug** — не выдумывать длинный текст; оставить минимальную заглушку и пометить slug в отчёте «нет контента».

## Куда класть контент в коде (обязательная карта)

### 1. Товары каталога (`/catalog/{slug}`)

| Что | Файл | Поле / компонент |
|-----|------|------------------|
| Данные товара (title, price, image) | `src/lib/products.ts` | `ALL_PRODUCTS` |
| SEO meta | `src/lib/product-seo.ts` | уже есть — **не перезаписывать** без явного указания |
| **Новый контент карточки** | **`src/lib/product-page-content.ts`** (создать) | `Record<slug, ProductPageContent>` |
| Страница | `src/app/catalog/[slug]/page.tsx` | рендер блоков из content + fallback |
| JSON-LD description | `src/lib/schema/product-page.ts` | брать `lead` или первый section из content, если есть |
| Чеклист slug | `docs/SAYT_KARTOCHKI_DLYA_KONTENTA.md` | сверка 39 позиций |

**Тип (создать):**

```ts
export type ProductPageContent = {
  /** 2–3 предложения под H1/ценой (видимый lead) */
  lead: string;
  /** Основные блоки ниже fold на desktop / под CTA на mobile */
  sections: {
    heading: string;
    body: string;
    list?: string[];
  }[];
  /** Опционально: компактная таблица характеристик */
  specs?: Record<string, string>;
};
```

### 2. Товары для мастеров (`/tovary-dlya-masterov/{slug}`)

| Что | Файл |
|-----|------|
| SEO meta | `src/lib/masters-product-seo.ts` |
| Длинное описание | расширить **`src/lib/masters-products.ts`** поле `description` **или** отдельный `masters-product-page-content.ts` по аналогии с каталогом |
| Страница | `src/app/tovary-dlya-masterov/[slug]/page.tsx` |

Для позиций с вариантами (слэбы, формы) — блок «Как выбрать размер» не дублирует прайс; отсылает к селектору на странице.

### 3. Листинги и хабы

| Страница | Файл контента | Минимум |
|----------|---------------|---------|
| `/catalog/vse-tovary` | `src/lib/catalog-shop-content.ts` или секция в `catalog-filters.ts` | **2 абзаца** intro + H2 «Как выбрать изделие» + ссылки на 6 SEO-разделов |
| `/catalog` | `src/lib/catalog-hub-content.ts` | **2 абзаца** + краткий список разделов |
| `/tovary-dlya-masterov` | `src/lib/masters-catalog-content.ts` | **2 абзаца** про ассортимент + H2 по группам (силикагель, заготовки, формы) |

Рендер: под H1, **над** сеткой товаров; стили как на SEO-лендингах (`text-sm sm:text-base leading-relaxed text-fg/70`).

### 4. SEO-лендинги категорий

Источник: `src/lib/seo-catalog-landings.ts` — **intro уже есть**.  
Дорабатывать только если во входном файле явно указан `landingKey` (breakfastTable, podnosy, …). Не копировать intro категории на карточки товаров дословно.

## Размещение на странице карточки товара (макет)

Порядок блоков **сверху вниз** (не менять без причины):

```
[Крошки / назад]
[Grid: фото | колонка текста]
  H1 (title из products.ts — не дублировать вторым H1)
  Цена
  Lead (2–3 предложения, уникальные)
  [Кнопки: корзина / WhatsApp]
  --- на mobile можно collapsible «Подробнее» только если lead + 1 section уже видны ---
[Ниже grid или full-width в Container]
  H2 sections[].heading + body (+ ul если list)
  H2 «Характеристики» — dl/grid из specs (если есть)
  H2 «Доставка и заказ» — короткий общий блок (1 раз в компоненте, не 39 копий в данных)
  Ссылка на родительскую категорию (SEO-лендинг) если slug в productSlugs
```

**Запрещено:**

- Скрывать основной текст через `display:none`, `sr-only`, табы без текста в DOM.
- Дублировать один и тот же абзац на все 39 карточек.
- Ставить H1 внутри sections (только H2/H3).
- Ломать `ProductCartActions`, `CatalogBackLink`, `JsonLd`.

**Компонент:** вынести в `src/components/shop/product-page-content.tsx` (presentational), данные — из lib.

## Требования к тексту (качество)

- Язык: **русский**, обращение на «вы», тон студии: спокойный, экспертный, без канцелярита.
- Цифры: «от X ₽» согласованы с `product.price` в коде.
- Гео: Краснодар / доставка по РФ — не чаще 1 раза на странице в теле (уже есть в footer).
- **Нет** вымышленных отзывов, рейтингов, «★★★★★», сертификатов, которых нет на сайте.
- Списки: маркеры через `<ul>`, не через `•` в одной строке (FAQ категорий — исключение, там `whitespace-pre-line`).
- Объём lead: **40–80 слов**; каждый section body: **60–120 слов**.

---

## SEO-согласованность

- Видимый **lead** должен раскрывать intent из **title** (`product-seo.ts`), но не повторять meta description символ в символ.
- В тексте 1–2 естественных вхождения тематики (например «столик для завтрака в постель», «часы из эпоксидной смолы») — без переспама.
- После добавления контента проверить: `Product.description` в JSON-LD = lead или сжатая версия lead (≤ 300 символов если нужно).
- Внутренние ссылки: на SEO-лендинг категории, `/catalog`, `/blog/{релевантная статья}` где уместно (1–2 ссылки на страницу).

## Порядок работ (для агента)

1. **Прочитать входной файл** заказчика; построить таблицу slug → есть/нет контента.
2. **Создать** `product-page-content.ts` (+ типы), заполнить из файла для каталога.
3. **Создать** `ProductPageContentView` и подключить в `catalog/[slug]/page.tsx`.
4. **Обновить** `product-page.ts` (schema): description из content.
5. **Листинги:** hub / vse-tovary / masters index — отдельные константы + JSX.
6. **Мasters:** контент в `masters-product-page-content.ts` или расширение `description`.
7. **Lint + `npm run build`** — без ошибок.
8. **Отчёт:** markdown-таблица slug | слов | статус; список slug без контента во входном файле.

---

## Критерии приёмки

- [ ] У каждой карточки каталога с контентом во входном файле ≥ **250 слов** видимого текста.
- [ ] Нет двух карточек с идентичным lead (>80% совпадения).
- [ ] Один H1, логичные H2, текст не скрыт от робота.
- [ ] Meta title/description **не изменены** (если не было отдельного ТЗ).
- [ ] JSON-LD Product.description обновлён там, где появился lead.
- [ ] `/catalog/vse-tovary` и `/catalog` — минимум **2 абзаца** уникального intro.
- [ ] Сборка проходит; мобильная вёрстка не ломается (проверить `/catalog/stol-river` и `/catalog/chasy-live`).
