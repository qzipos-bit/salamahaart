# Промт: визуальное оформление категории «Столик для завтрака»

Используйте для дизайнера, Figma или ИИ-генерации макета.

## Контекст бренда
Премиальный ручной декор из эпоксидной смолы и дерева: спокойные зелёно-кремовые тона сайта, шрифты Cormorant (заголовки) + Manrope (текст), мягкие тени и «стеклянные» панели (`glass-panel`).

## Цель страницы
Страница категории `/catalog/dekorativnye-stoliki/breakfasttable`: доверие, понятный оффер, лёгкий путь к заказу в WhatsApp и к карточкам товаров.

## Структура блоков (сверху вниз)
1. **Хлебные крошки** — компактно, вторичный цвет текста; последний пункт без ссылки.
2. **Hero категории** — H1 + короткий подзаголовок (опционально) + **карточка-интро** (светлый фон, бордер): два абзаца о пользе столика в постель/на диван и про заказ на изготовление.
3. **Секция «Коллекция»** — H2, сетка **6 карточек** товара (2 колонки планшет, 3 — десктоп), единый стиль с остальным каталогом.
4. **FAQ** — H2 «Часто задаваемые вопросы», аккордеон `<details>`/`<summary>`: вопрос крупнее, ответ с нормальными отступами; списки в ответах — маркированные, без перегруза.
5. **Нижний блок** — ссылки на другие тематические разделы + «Полный каталог».

## UX
- На мобиле: одна колонка карточек, FAQ на всю ширину, CTA WhatsApp заметен после пустой витрины (если нет товаров) — для этой категории товары есть.
- Избегать «стены текста»: интро 2 абзаца, FAQ — раскрывающиеся блоки.

## Референсы на код
Реализация в `src/components/catalog/seo-catalog-landing-view.tsx`, данные — `src/lib/seo-catalog-landings.ts`, FAQ — `src/lib/breakfast-table-faq.ts`.

## Технические ограничения
Next.js App Router, Tailwind 4, без тяжёлых анимаций; изображения товаров — `next/image`, соотношение карточек как в `ProductCard`.
