# Промпт для UX/UI (Figma) + handoff в разработку (Next.js)

**Назначение:** единый бриф для дизайнера / AI-генерации макетов и для разработчика. Формат: Desktop + Mobile, Auto Layout, компонентная система, готовность к передаче в код.

**Бренд-референс:** Salamaha Fine Art — личный бренд художника по эпоксидной смоле (Victoria, resin artist). Не корпорация: тёплый голос, доверие к мастеру, премиальный handmade.

---

## 1. Северная звезда (визуальный DNA)

Сверяй все решения с референсами бренда (логотип с кругом, веткой и волной; спил дерева с годовыми кольцами; фоны с мягкими «волнами» смолы/чернил; палитра Instagram-профиля).

**Ассоциации:** Pinterest-эстетика, стекло и свет, прозрачность, уют + природа, «тихая» роскошь, галерея, а не маркетплейс.

**Запрещённый тон:** яркие кислотные цвета, перегруз текстом, шаблон «универсального лендинга», ощущение дешёвого маркетплейса.

## 2. Дизайн-токены (цвет, типографика, эффекты)

### Цвета
| Роль | Значение / описание |
|------|---------------------|
| Фон основной | `#F8F8F6` — молочный off-white, лёгкая «бумажная» текстура допустима |
| Текст основной | `#1A1A1A` — тёмно-серый, не чистый чёрный |
| Акцент зелёный (primary UI) | Глубокий лесной для текста логотипа и сильных акцентов: ориентир `#2D4033` / `#1E3024` |
| Акцент зелёный (мягкий) | Пыльная шалфей / оливково-зелёный для фонов блоков, бейджей, подложек — как на референсах с кругом и wood grain |
| Обводка / «крем» | Тёплый молочно-белый для колец, тонких рамок: ориентир `#F8F8F6` или чуть теплее при контрасте с зелёным |
| Золото | Минимально: тонкие линии, акцент на одной волне/разделителе, hover-искра — не заливки больших площадей |

### Типографика
- **Заголовки (H1–H3):** современный serif, премиальное ощущение, крупные размеры, у H2/H3 — умеренный letter-spacing для «галереи».
- **Подзаголовки логотипного типа:** лёгкий script / рукописный акцент только точечно (имя бренда, подпись блока «О мастере») — не злоупотреблять читаемостью.
- **Body / UI:** чистый sans-serif, нейтральный, хорошая иерархия (размер / вес / цвет).

### Эффекты и форма
- **Glassmorphism:** полупрозрачные панели (header, CTA-блоки, карточки при необходимости), blur умеренный, читаемость текста обязательна.
- **Тени:** мягкие, диффузные, без жёсткого чёрного.
- **Скругления:** карточки и кнопки **16–24px**, согласованно по системе.
- **Воздух:** вертикальные ритмы **80–120px** между крупными секциями на desktop; на mobile пропорционально уменьшить, но не сжимать до «плотного шаблона».

### Текстура
- Лёгкий noise / зерно на фоне (очень низкая непрозрачность) — отсылка к бумаге и органике референсов.

## 3. Сетка и layout

- **12 колонок**, **max-width контейнера 1200–1320px**, выравнивание по центру.
- Крупные поля (gutter) и внутренние отступы секций согласованы с модульной сеткой.
- **Mobile:** одна колонка для карточек, сохранение визуальной иерархии; **sticky** primary-действие «Написать» / мессенджер (см. контакты).

---

## 4. Структура экранов (обязательные блоки)

1. **Hero** — крупное фото/видео изделия; H1: «Авторские изделия из эпоксидной смолы»; подзаголовок; **Primary CTA:** «Смотреть каталог»; **Secondary:** «Заказать индивидуально»; ряд мини-преимуществ (иконки + короткий текст).
2. **About** — живое фото мастера; текст от первого лица; акценты «10+ лет», «ручная работа»; карточки достижений / фактов.
3. **Categories** — 4–6 карточек (Часы, Столы, Картины, Декор, Посуда и т.д.); hover: лёгкое затемнение + название.
4. **Featured products** — фото, название, цена, кнопка; бейджи: «Хит», «-30%», «Новинка».
5. **Advantages** — 4–6 пунктов, минималистичные иконки/иллюстрации, короткие подписи.
6. **Custom order** — визуально выделенный блок (фон или glass); H2: «Создам изделие под ваш интерьер»; поля имя + телефон; CTA «Оставить заявку».
7. **Gallery / Portfolio** — masonry, крупные изображения, hover zoom (лёгкий scale).
8. **Reviews** — карточки с цитатой, именем, при возможности фото/аватар; опционально визуальная отсылка к мессенджеру (аккуратно, без кича).
9. **Courses / Masterclass** — карточки онлайн/оффлайн, CTA «Записаться» (связь с направлением школы/курсов смолы+дерево).
10. **Blog preview** — 3–4 карточки статей.
11. **Delivery / Payment** — иконки + краткий текст.
12. **Contacts** — телефон, **выделенная кнопка WhatsApp**, адрес, карта (или embed placeholder).

**UX-фишки (заложить в макет / прототип):**
- Мини-квиз «Подберём декор за 2 минуты» (entry point в hero или плавающая кнопка).
- Быстрая заявка (короткая форма / repeat блока custom order).
- Фокус на визуале, минимум текста, максимум эмоции.

---

## 5. Компонентная система (обязательные компоненты Figma)

С вариантами состояний: default, hover, active, disabled (где уместно).

- Кнопки: **primary** (зелёный акцент), **secondary** (outline / ghost / glass), **текстовая** при необходимости.
- Карточка товара (с бейджами).
- Карточка категории.
- Поля формы + состояния ошибки/успеха.
- Бейджи: хит, скидка, новинка.
- Header (лого, навигация, корзина/иконка магазина, CTA контакт).
- Footer (ссылки, соцсети, юридические короткие строки).
- Модальное окно / drawer для квиза или быстрой заявки (хотя бы один вариант).

---

## 6. Микроанимации (спецификация для dev handoff)

- Hover на карточках: лёгкий подъём или смена тени + затемнение оверлея на категориях.
- Плавное появление секций при скролле (в дизайне — заметки для разработчика; в прототипе — по желанию smart animate).
- Scale изображений в галерее при hover (1.02–1.05, без клиппинга некрасивого).

---

## 7. Цели и критерии качества

Сайт должен вызывать: **«хочу купить»**, **«красиво и уникально»**, **«доверяю мастеру»**.

Уровень результата: презентабельно для **Behance / Awwwards** как визуальная подача; технически — чистая система для переноса в **Next.js** (именованные стили, токены, повторяемые компоненты).

## 9. Краткая инструкция для разработчика (Next.js)

После утверждения макета: вынести токены в CSS variables или theme; собрать секции как изолированные компоненты; изображения — `next/image`; формы — с валидацией и явной отправкой (endpoint/Telegram/WhatsApp link по брифу заказчика); masonry — через CSS grid или библиотеку с учётом CLS; доступность: контраст текста, фокус-стили, alt у изображений.
